DETAILED GUIDE: INTEGRATING AWS SMALL HOUSE WORLD INTO CLEARPATH SIMULATOR

This guide provides the complete sequence of operations to install, configure, and run the AWS RoboMaker Small House World within the Clearpath Gazebo Sim environment.

1. INITIAL REPOSITORY SETUP
---------------------------
Navigate to the root of the ROS2 workspace and clone the repository:
$ cd ~/clearpath_ws
$ git clone -b ros2 https://github.com/aws-robotics/aws-robomaker-small-house-world.git src/aws-robomaker-small-house-world

2. MODEL PREPARATION & FIXES
----------------------------
A. Update Model URIs:
Convert internal model references to use the model:// URI scheme:
$ find src/aws-robomaker-small-house-world/models -name "model.sdf" -exec sed -i 's|file://models/|model://|g' {} +

B. Fix Invalid Inertias & Static Status:
Gazebo Sim requires stable inertias for dynamic objects. For residential furniture, it is recommended to strip inertia blocks and force <static>true</static> to ensure simulation stability.

C. Fix Texture Paths & Synchronize Photos:
- Update texture paths in .DAE files from '../../../../photos/' to '../../../photos/'.
- IMPORTANT: The small house world shares many models with the hospital world. If both are installed, synchronize their 'photos' directories to ensure Gazebo can resolve textures regardless of which package it finds the model in first.
$ cp -n src/aws-robomaker-small-house-world/photos/* src/aws-robomaker-hospital-world/photos/
$ cp -n src/aws-robomaker-hospital-world/photos/* src/aws-robomaker-small-house-world/photos/

3. WORLD CONVERSION (.world to .sdf)
------------------------------------
Convert the original .world file to Gazebo Sim compatible .sdf format in src/clearpath_simulator/clearpath_gz/worlds/small_house.sdf. Include the standard plugins and spherical coordinates.

4. LAUNCH CONFIGURATION UPDATES
-------------------------------
Edit src/clearpath_simulator/clearpath_gz/launch/gz_sim.launch.py to include the house resource paths:
- The package root: src/aws-robomaker-small-house-world
- The models directory: src/aws-robomaker-small-house-world/models
- The photos directory: src/aws-robomaker-small-house-world/photos

5. BUILDING
-----------
Build the workspace:
$ colcon build --packages-select clearpath_gz aws_robomaker_small_house_world --symlink-install --allow-overriding clearpath_gz

6. RUNNING THE SIMULATION
-------------------------
Source the workspace and launch:
$ source install/setup.bash
$ ros2 launch clearpath_gz simulation.launch.py world:=small_house
